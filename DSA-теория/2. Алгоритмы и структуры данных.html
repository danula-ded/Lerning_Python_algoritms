  • Алгоритм. Память и время как ресурсы.

    Свойства алгоритма

    • конечность — алгоритм всегда должен заканчиваться после выполнения конкретного числа шагов
    • ввод — алгоритм имеет некоторое (возможно, равное нулю) число входных данных
    • вывод — алгоритм имеет некоторое (больше нуля) число выходных данных, то есть данных, имеющих определённую связь с входными данными
    • определённость — все действия алгоритма чётко и недвусмысленно определены (не возникают разногласия о них)
    • эффективность — все действия алгоритма достаточно просты, их можно легко выполнить с помощью карандаша и бумаги

    # ставим большее число первым, а меньшее вторым
    # (меняем местами или оставляем)
    if first < second:
        first, second = second, first
    # пока меньшее (второе) число не равно 0 - продолжаем

    def gcdExtended(first: int, second: int) -> tuple[int, int, int]:
    # начало без изменений (обычный алгоритм Евклида)
    if second == 0:
        # только возвращаем дополнительно коэффициенты
        # 1 для ненулевого first и 0 для нулевого second
        return first, 1, 0
    # получаем делитель и коэффициенты предыдущего рекурсивного вызова
    gcd, x_prev, y_prev = gcdExtended(second, first % second)
    # обновляем коэффициенты для текущего вызова
    x = y_prev - (first // second) * x_prev
    y = x_prev
    return gcd, x, y

    def sieveOfEratosthenes(n):
    # создаём список, в котором будем хранить
    # зачёркнуто число (непростое, False) или нет (простое, True)
    prime = [True for _ in range(n+1)]

    start = 2
    # так как делителей после корня из числа нет

  • Криптография: схемы с закрытым ключом, RSA.

    Алгоритм работы

    алгоритм основан на двух неразрывно связанных ключах:
    -
    открытом, передаётся по открытым каналам связи, им шифруют сообщения
    -
    закрытом, держится в секрете и никуда не передаётся, им дешифруют сообщения
    ситуация: Боб хочет передать Алисе сообщение, но лично он это сделать не может, поэтому необходимо использовать посредника, например, Стива. Боб передаёт Алисе информацию о сюрпризе на день рождения Стива, так что не может допустить, чтобы Стив увидел это сообщение
    1. перед обменом сообщением, Боб просит у Алисы её открытый ключ
    1. после получения ключа, переданного через Стива, Боб шифрует своё сообщение ключом Алисы
    1. Боб передаёт Алисе через Стива зашифрованное сообщение
    1. Алиса расшифровывает сообщение своим закрытым ключом

    таким образом, Стив видел открытый ключ Алисы и зашифрованное сообщение от Боба, но без закрытого ключа Алисы он не сможет расшифровать сообщение и узнать его содержимое

    но в этой ситуации Стив может схитрить:

    1. Боб также просит у Алисы её открытый ключ
    1. Стив получает ключ, но передаёт Бобу свой ключ вместо ключа Алисы
    1. Боб шифрует своё сообщение полученным ключом и передаёт Стиву
    1. Стив не сразу передаёт сообщение Алисе, а расшифровывает его, читает и шифрует обратно уже ключом Алисы
    1. Алиса расшифровывает полученное сообщение своим закрытым ключом

    таким образом, Боб и Алиса ничего не заметят, а Стив прочитает сообщение

    такая атака называется «человек посередине» или MITM, Man In The Middle

    закрытый ключ может не только шифровать, но и подписывать данные ⇒ данные нельзя будет подменить

    решением этой проблемы будет «цепочка доверия» или Chain of Trust:

    1. попросим Гранта, которому доверяем, подписать открытые ключи Алисы и Боба

    теперь Стив не сможет подменить ключ Алисы, ведь Боб сразу заметит подмену

    1. попросим Гранта подписать открытый ключ Марку, а его уже подписать открытые ключи Алисы и Боба ⇒ создадим «цепочку доверия»

def bubble_sort(nums: list) -> None:
    # переменная, говорящая "изменили ли за последний перебор" список,
    # то есть отсортирован он (ничего не меняли, False) или нет (True)  
    swapped = True
    # пока список не отсортирован (пока True)
    while swapped:
        # изначально ставим swapped в False,
        # то есть не меняли список в текущем переборе
        swapped = False
        # проходимся (перебираем) по всем элементам списка
        # -1, так как для последнего элемента нет "следующего", 
        # его сравним с предпоследним
        for ind in range(len(nums) - 1):
            # если элементы (текущий и следующий) стоят не в том порядке
            # (не по возрастанию в данном случае)
            if nums[ind] > nums[ind + 1]:
                # то меняем их местами
                nums[ind], nums[ind + 1] = nums[ind + 1], nums[ind]
                # поменяли список => изменяем переменную swapped
                swapped = True

        visited.add(v)
        curr_way.add(v)
        # перебираем все смежные вершины с вершиной v
        for neighbour in self.graph[v]:
            # если смежная вершина не пройдена, то проходимся по её "ветке"
            if neighbour not in visited:
                # возвращаем True, если вызов вернул True (в "ветке" цикл)
                if self.isCyclicUtil(neighbour, visited, curr_way) == True:
                    return True
            # если мы были в смежной вершине по текущего пути, то это цикл
            elif neighbour in curr_way:
                return True
        # если дошли сюда, то у "ветки" вершины v нет циклов, 
        # значит пойдём по другому пути (убираем v из текущего пути)
        # и возвращаем False для этой "ветки" (нет циклов)
        curr_way.pop(v)
        return False
 
    # сама функция проверки наличия циклов в графе
    def isCyclic(self) -> bool:
        # создаём множества (для скорости) пройденных вершин и текущего пути
        visited = set()
        curr_way = set()
        # перебираем все вершины
        for v in range(len(self.graph)):
            # если вершина не пройдена - проверяем её "ветку" на цикл
            if v not in visited:
                # возвращаем True, если доп. функция вернула True (есть цикл)
                if self.isCyclicUtil(v, visited, curr_way):
                    return True
        # если доп. функция ни разу не вернула True, то цикла нет (False)
        return False

    позволяет сделать топологическую сортировку или найти циклы (при них сортировка невозможна)
    1. вычисляем количество входящих рёбер для каждой вершины
    1. добавляем в очередь вершины с 0 входящими рёбрами (дугами)
    1. берём первую вершину из очереди, добавляем её в список отсортированных вершин и увеличиваем счётчик посещённых вершин на 1

    1. если количество входящих рёбер смежной вершины стало 0, то добавляем её в очередь
    1. если очередь не пустая — возвращаемся к шагу 3
    1. если количество вершин в графе и количество посещённых вершин не совпало — в графе цикл

# граф в виде списка смежности
class Graph:
    def __init__(self) -> None:
        # список смежности в виде словаря
        self.graph = defaultdict(list)

    # добавление нового ребра из v1 в v2, v1 и v2 - номера вершин
    def add_edge(self, v1: int, v2: int) -> None:
        self.graph[v1].append(v2)

    # сама функция топологической сортировки
    def topologicalSort(self) -> None:
        # создаём список с количеством входящих рёбер для каждой вершины
        in_degree = [0] * len(self.graph)
        # заполняем список in_degree, проходясь по графу
        for v1 in self.graph:
            for v2 in self.graph[v1]:
                in_degree[v2] += 1
        # создаём очередь и заполняем её вершинами с 0 входящими рёбрами
        zeros = deque()
        for v in range(len(self.graph)):
            if not in_degree[v]:
                zeros.append(v)
        # создаём счётчик посещённых вершин и список-результат сортировки
        visited = 0
        top_order = []
        # перебираем всю очередь
        while zeros:
            # вытаскиваем вершину из начала очереди и добавляем в результат
            v = zeros.popleft()
            top_order.append(v)
            # проходимся по смежным вершинам с вершиной v
            for neighbour in self.graph[v]:
                # уменьшаем количество входящих рёбер (прошли их "родителя")
                in_degree[neighbour] -= 1
                # если количество стало = 0, то добавляем вершину в очередь
                if not in_degree[neighbour]:
                    zeros.append(neighbour)
            # прошли вершину, добавляем 1 к счётчику посещённых вершин
            visited += 1
        # если количество посещённых вершин и вершин в графе не совпадает,
        # то в графе есть цикл (дважды уменьшили на 1, поэтому не совпадает)
        if visited != len(self.graph):
            print ('В графе есть цикл!')
        # иначе цикла нет, а значит выводим результат сортировки
        else:
            print(top_order)
