Основы программирования

  • Git, продвинутые команды.
    # здесь приведены самые частые команды и флаги, но далеко не все
# чтобы узнать больше, можно использовать встроенную помощь

# помощь по командам git
git --help
# помощь по конкретной команде git
git status -h
    • git config — настройка git
      # настройка глобального (распространяющегося на все проекты) имени
git config --global user.name "Albert"
# настройка глобального адреса электронной почты
git config --global user.email albert@gmail.com
# просмотр текущих настроек
git config --list
    • git clone — клонирование удалённого репозитория по ссылке
      # пример ссылки - https://github.com/test.git
git clone ссылка
    • git init — инициализация пустого репозитория git в текущей папке
    • git status — текущее состояние репозитория: индексированные, неиндексированные файлы и текущая ветка
    • git add — добавить файлы/папки в репозиторий (сделать индексированными)
      # добавить конкретный файл
git add file.txt
# добавить все файлы
git add .
git add -A
    • git commit — сохранение/фиксация изменений (ТОЛЬКО в индексированных файлах), текущего состояния репозитория. У каждого коммита есть идентификатор и название.
      # лучше и удобнее сразу указать название
git commit -m "название коммита"
# можно изменить название последнего коммита
# и дополнить его изменениями в отслеживаемых файлах на момент дополнения
git commit --amend -m "новое название коммита"
    • git remote — управление ссылками на удалённые репозитории (например, на Github). Используются для упрощения: в командах можно писать название, а не ссылку целиком. При клонировании репозитория устанавливается автоматически.
      # для основной ссылки часто используют название origin
git remote add название ссылка
# также ссылку можно удалить
git remote remove название
    • git push — загрузка коммитов в удалённый репозиторий в конкретную ветку
      git push ссылка ветка
git push название_ссылки ветка
# установка ссылки и ветки по умолчанию, далее можно писать git push
git push -u название_ссылки ветка
    • git pull — скачивание изменений из удалённого репозитория
      git pull ссылка
git pull название_ссылки

# сделает не обычный merge, если локальная и удалённые версии расходятся,
# а rebase (то есть коммиты допишутся в конец)
git pull --rebase
    • git log — просмотр истории всех коммитов в текущем репозитории
      # можно по-разному форматировать с помощью флагов
git log
# в виде графа, показывают схему ветвлений (веток)
git log --graph

# короткая версия git log --graph --oneline --decorate
git history
    • git show — просмотр определённого коммита по его идентификатору
      # можно писать полный идентификатор
git show 9e0f44c349a3acedba94bfe85ba47e075ea52fbb
# или краткую версию (первые несколько символов)
git show 9e0f44c3
    • git diff — сравнение изменений между двумя коммитами по идентификаторам
      # вместо id1 и id2 - идентификаторы коммитов 
git diff id1..id2
    • git branch — управление ветками
      # проверка текущей ветки
git branch
# создание новой ветки с названием
git branch название
# удаление ветки
git branch -d название
    • git checkout — переход в другую ветку
      # переход в существующую ветку
git checkout название
# создание новой ветки и переход в неё
git checkout -b название
    • git merge — слияние веток
      # слияние ветки с названием в текущую
git merge название
    • git revert — откат коммитов с помощью создания новых, в которых откатываются изменения. Обычно используют в публичных ветках, где работу ведут несколько человек.
      git revert 9e0f44c3
# HEAD - ссылка на последний коммит (если не изменяли)
git revert HEAD
    • git reset — откат коммитов с помощью их удаления. Есть 3 режима
      # soft
# коммит удаляется, а его изменения попадают в отслеживаемые
git reset --soft 9e0f44c3
# переход к состоянию прошлого коммита (на 1 назад от текущего HEAD)
git reset --soft HEAD~1
# mixed, по умолчанию
# коммит удаляется, а его изменения попадают в неотслеживаемые
git reset 9e0f44c3
# hard
# коммит удаляется, а его изменения пропадут
git reset --hard 9e0f44c3
    • git rm — удаление файлов из индексированных, а также на диске
      # файл удалится и с индексированных, и с диска
git rm file.txt
# файл удалится с индексированных, но останется на диске
git rm --cached file.txt
    • git rebase — объединение изменений двух веток с перезаписью истории ветки
      # обычный режим, берёт коммиты из текущей ветки и применяет их
# в конец переданной ветки
git rebase ветка
# открывает интерактивный режим
# (изменение самих коммитов, их порядка и информации в них)
# открывается текстовый файл (как в merge конфликте)
# с подсказками по применению
git rebase -i
# можно указать до какого коммита открыть режим, например HEAD~6

# нужно быть осторожным, так как переписывается история
# отменить rebase можно с помощью git reflog
    • git stash — сохраняет текущие индексированные и неиндексированные изменения, при этом удаляя их из репозитория, потом их можно восстановить
      # сохранение изменений (2 команды с одинаковым смыслом)
git stash
# ... после выполнения нескольких команд
# применение изменений, удаляются из набора изменений
git stash pop
# применение изменений, не удаляются из набора изменений
git stash apply
  • Основные типы данных Python, классификация типов данных,
    комментарии. Хранение базовых типов данных в памяти.

    Есть 2 вида комментариев:

    # однострочные, выделяются октоторпом/шарпом 
print('Hello world')

# многострочные
# выделяются октоторпом/шарпом на каждой строчке
print('Hello world')

"""
Также многострочные комментарии
можно выделять тройными кавычками (одинарными или двойными)
"""
print('Hello world')

    Данные в переменных и константах могут быть разных типов

    • Пустой (nonetype) None — пустота
    • Булевый тип (bool) True/False— истина или ложь
    • Множества (set) {1, 'abc', 3.0} — неупорядоченный набор уникальных (дубликаты будут удалены) элементов, можно объединять, пересекать и вычитать
    • Словари (dict) {'a': 1, 2: 'b'} — неупорядоченный набор пар ключ-значение, по ключу можно получить значение
    • Последовательности — объединение нескольких элементов, к ним можно обращаться по индексам и срезам
      • строки (string) 'a1b2c3' — последовательность символов Юникода, могут быть однострочные и многострочные (выделяются тройными кавычками)
      • списки (list) [1, 'abc', 3.0, 1] — упорядоченная последовательность элементов разных типов данных
      • кортежи (tuple) (1, 'abc', 3.0, 1) — неизменяемая версия списков, поэтому работают быстрее
    • Числа
      • целые (int) 21 — любой длины
      • с плавающей точкой (float) 6.0 — с точностью до 15 знаков после запятой
      • комплексные (complex) 1 + 2j — в формате действительная часть + мнимая (j)
    # так как в Python динамическая типизация, тип данных переменной
# можно преобразовывать с помощью встроенных функций
number = 1.0
print(type(number)) # <class 'float'>
number = int(number)
print(type(number)) # <class 'int'>

    Классификация типов данных

    • Изменяемые (mutable) — можно изменить переменную с таким типом данных
      • списки
      • словари
      • множества
    • Неизменяемые (immutable) — нельзя изменить переменную с таким типом данных
      • числа
      • булевые значения
      • строки
      • кортежи
    # адрес памяти не изменяется при изменении списка
# по факту изменяем данные в текущем месте памяти
test_list = [1, 2, 3, 4]
print(id(test_list)) # 4338023104
test_list.append(5)
print(id(test_list)) # 4338023104

# адрес памяти изменяется при изменении переменной с числом
# по факту занимаем новое место в памяти и даём ему старое название
test_number = 5
print(id(test_number)) # 4348568504
test_number = 55
print(id(test_number)) # 4348570104

    Также можно классифицировать типы данных как:

    • Индексируемые — можно обращаться к части объекта этого типа данных по индексу

      пример: список, строка
    • Неиндексируемые — нельзя обращаться к части объекта этого типа данных по индексу

      пример: число, булевое значение
    • Содержащие уникальные элементы
      пример: множество
    • Не содержащие уникальные элементы
      пример: списки, кортежи

    Хранение базовых типов данных в памяти

    • булевы значения bool — занимают 28 байт
    • целые числа int — занимает 28 байт + дополнительные 4 байта за каждые 2302^{30}230﻿
    • вещественные числа float — хранение описано стандартом IEEE-754, занимают 24 байта
    • строки str — хранится как массив символов Unicode, занимает 49 байт + дополнительные байты за каждый символ
    • список list — занимает 56 байт + 8 байт за каждый элемент + вес всех элементов
    • кортеж tuple — занимает 40 байт + 8 байт за каждый элемент + вес всех элементов
    • множество set — занимает 216 байт + неточное кол-во байт за некоторое количество элементов (+152 за первые 5 элементов, +256 за следующие 3 и т.п.) + вес всех элементов
    • словарь dict — занимает 64 байта + неточное кол-во байт за некоторое количество пар + вес всех ключей и значений
  • Строки, синтаксис и операции со строками.

    Создание строк

    # с помощью одинарных или двойных кавычек
# можно использовать один тип кавычек внутри другого
my_string = 'Hello'
print("Book 'Hello'")  # Book 'Hello'

# длинные и многострочные строки создают с помощью
# тройных кавычек (любого типа) или символа переноса \
my_string = '''Hello, World!
						I will take the exam.'''
my_string = """Hello, World!
						I will take the exam."""
my_string = 'hello \
						world'

# удалить строку можно с помощью del
del my_string

    Операции со строками

    • индексы (порядковые номера символов) и срезы (взятие части строки индексами)
      my_string = 'Hello'

# можно взять символ по индексу, в том числе и отрицательному
print(my_string[0])          # H
print(my_string[-1])         # o

# индекс должен быть корректным целым числом
print(my_string[1.0])        # TypeError, не целое число
print(my_string[10])         # IndexError, последний индекс - 4

# можно найти индекс символа
print(my_string.index('H'))  # 0

# срез (взятие части строки) выглядит как начало:конец:шаг
print(my_string[1:4])        # ell 

# нельзя изменить/удалить символ по индексу/срезу, только строку целиком
# так как строки неизменяемы
my_string[4] = 'l'           # TypeError
my_string[1:4] = 'ooo'       # TypeError
del my_string[4]             # TypeError
    • управляющие последовательности (escape-последовательности)
      • \\ — обратный слэш
      • \’ — одинарная кавычка
      • \" — двойная кавычка
      • \n — перенос строки
      • \t — горизонтальная табуляция
      • \v — вертикальная табуляция
      # можно экранировать (сделать обычным символом) кавычки
print('Тестирование'кавычки')      # Error, кавычка закрыла строку
print('Тестирование\'кавычки')     # Тестирование'кавычки

# игнорирование управляющих последовательностей
print(r'Тестирование\n переноса')  # Тестирование\n переноса
    • интерполяция и форматирование
      string1 = 'Hello'
string2 = 'World'

# интерполяция (f-строки)
print(f'{string1} big {string2}')                  # Hello big World

# форматирование по позиционным и ключевым аргументам
print('{} big {}'.format(string1, string2))        # Hello big World
print('{h} big {w}'.format(w=string2, h=string1))  # Hello big World
# форматирование чисел с плавающей точкой
print('{:1f}'.format(1.18))                        # 1.2

# форматирование по центру, левому и правому краю
# строка дополняется определёнными символами до определённой длины
# также можно форматировать с помощью f-строк
print(f'{string1:+<9}')                            # Hello+++
print('{:+<9}'.format(string1))                    # Hello++++
print('{:+^9}'.format(string1))                    # ++Hello++
print('{:+^8}'.format(string1))                    # +Hello++
print('{:+>9}'.format(string1))                    # ++++Hello
# если символов самой строки больше - ничего не меняется
print('{:+^3}'.format(string1))                    # Hello
    • методы
      my_string = 'Hello'
my_list = ['H', 'e', 'l', 'l', 'o']

# разбиение и сбор строки по разделителю
print(my_string.split('e'))         # ['H', 'llo']
print('-'.join(my_list))            # H-e-l-l-o

# перевод в различные регистры
print(my_string.lower())            # hello
print(my_string.upper())            # HELLO

# нахождение подстроки (части строки), возвращает индекс
print(my_string.find('el'))         # 1, ищет откуда начинается
print(my_string.find('l'))          # 2

# проверка состава строки
print(my_string.isalpha())          # True, все символы из алфавита
print(my_string.islower())          # False, все символы в нижн. регистре

# замена шаблона в строке
print(my_string.replace('H', 'm'))  # mello
    • остальные операции
      string1 = 'Hello'
string2 = 'World'

# конкатенация (объединение нескольких строк в одну) и умножение
print(string1 + string2)     # HelloWorld
print(string1 * 3)           # HelloHelloHello

# взятие длины
print(len(string1))          # 5

# сравнение (по длине)
print(string1 == string2)    # True

# проверка на вхождение
print('l' in string1)        # True

# итерирование по строке
for letter in string1:
		print(letter, end='_')   # H_e_l_l_o
  • Списки. Операции со списками. Кортежи.

    Создание списков

    # пустого
my_list = []
my_list = list()

# со значениями
my_list = [1, 'hello', 3, True, (1, 5)]
my_list = list('cat')         # ['c', 'a', 't']
my_list = '1/2/3'.split('/')  # ['1', '2', '3']

# списки могут быть вложенными
my_list = [1, [2.0, 'world', 4], False]

    Операции со списками

    • индексы (порядковые номера элементов) и срезы (взятие части списка индексами)
      my_list = ['H', 'e', 'l', 'l', 'o']
my_nested_list = ['H', ['e', 'l', 'l'], 'o']

# можно взять элемент по индексу, в том числе и отрицательному
print(my_list[0])               # H
print(my_list[-1])              # o

# также во вложенных списках
print(my_nested_list[1])        # ['e', 'l', 'l']
print(my_nested_list[1][0])     # e

# индекс должен быть корректным целым числом
print(my_list[1.0])             # TypeError, не целое число
print(my_list[10])              # IndexError, последний индекс - 4

# можно найти индекс элемента или количество повторений по его значению
print(my_list.index('H'))       # 0
print(my_list.count('l'))       # 2

# срез (взятие части списка) выглядит как начало:конец:шаг
print(my_list[1:4])             # ['e', 'l', 'l']

# можно изменить/удалить элемент по индексу/срезу
# так как списки изменяемы
my_list[4] = 'l'                # ['H', 'e', 'l', 'l', 'l']
my_list[1:4] = ['o', 'o', 'o']  # ['H', 'o', 'o', 'o', 'o']
del my_list[4]                  # ['H', 'e', 'l', 'l']
del my_list[1:4]                # ['H', 'o']
    • добавление элементов
      my_list = [1, 2, 3]
# один элемент в конец
my_list.append(4)         # [1, 2, 3, 4]
my_list.append([5, 6])    # [1, 2, 3, 4, [5, 6]]

my_list = [1, 2, 3]
# несколько элементов в конец
# добавляем обязательно список, даже если из одного элемента
my_list.extend([4])       # [1, 2, 3, 4]
my_list.extend([5, 6])    # [1, 2, 3, 4, 5, 6]
my_list += [7]            # [1, 2, 3, 4, 5, 6, 7]
my_list += [8, 9]         # [1, 2, 3, 4, 5, 6, 7, 8, 9]

my_list = [1, 2, 3]
# один элемент по индексу
my_list.insert(1, 4)      # [1, 4, 2, 3]
my_list.insert(0, [0, 5]) # [[0, 5], 1, 4, 2, 3]
    • удаление элементов
      my_list = [1, 3, 2, 3, 4, 5]
# по индексу, удалённый элемент возвращается
el = my_list.pop()  # [1, 3, 2, 3, 4]
print(el)           # 5, удалился последний элемент
my_list.pop(0)      # [3, 2, 3, 4]

# по значению, удалённый элемент не возвращается
my_list.remove()    # TypeError, не указано что удалять
my_list.remove(3)   # [2, 3, 4]

# всех элементов
my_list.clear()
print(my_list)      # [], список очистился
del my_list
print(my_list)      # NameError, список удалился
# также с помощью del можно удалять по индексам и срезам (см. выше)
    • изменение порядка элементов
      my_list = [3, 5, 1, 2, 4]

# переворачивание списка
my_list.reverse()   # [4, 2, 1, 5, 3], изменяет исходный список
reversed(my_list)   # [4, 2, 1, 5, 3], возвращает новый список

# сортировка списка по возрастанию и убыванию
my_list.sort() # [1, 2, 3, 4, 5], изменяет исходный список
sorted(my_list) # [1, 2, 3, 4, 5], возвращает новый список
my_list.sort(reverse=True)  # [5, 4, 3, 2, 1]

# сортировка списка по определённому параметру
new_list = [(1, 2), (3, 1), (4, 5)]
# по второму значению в кортежах
new_list.sort(key=lambda pair: pair[1]) # [(3, 1), (1, 2), (4, 5)]
    • остальные операции
      list1 = [1, 2, 3]
list2 = [4, 5, 6]
list3 = [2, 3, 1]

    # функция-генератор, содержит yield
def range_(first=0, last=5, step=1):
		number = first
		# лучше использовать while, так как range - генератор,
		# а генератор в генераторе - не очень хорошая идея
		while number < last:
				# yield похож на return, но если
				# return отдаёт значение и завершает функцию 
				# (можно только начать её сначала)
				# то yeild отдаёт значение и не завершает функцию
				# (продолжается с того же места при повторном вызове)
				yield number
				number += step
